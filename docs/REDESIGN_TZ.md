# Аналіз та ТЗ: Редизайн і розширення PivdenShop

---

## Частина 1. Порівняльний аналіз з українськими маркетплейсами

Порівнюю з Rozetka, Prom.ua, OLX, Allo.ua, Epicentr — лідерами українського e-commerce.

### Що є у конкурентів і відсутнє у PivdenShop

#### Головна сторінка:

- Немає банерного слайдера (carousel) з акціями та промо-пропозиціями
- Немає секції "Акції/Знижки" та таймера зворотного відліку на акції
- Немає секції "Нові надходження"
- Немає секції "Рекомендовано для вас" (персоналізація)
- Немає блоку категорій з іконками та візуальним представленням
- Немає банера "Стати продавцем" для залучення продавців
- Немає секції довіри: "Безпечна оплата / Швидка доставка / Гарантія якості" з іконками
- Немає блоку "Топ продавці тижня"

#### Візуал і стилі:

- Темна тема як єдина — немає перемикача light/dark mode
- Картки товарів без hover-анімацій (немає підйому картки, тіні, появи кнопки "В кошик" при hover)
- Немає skeleton loading (завантаження показує порожні блоки)
- Немає lazy loading з анімацією для зображень
- Логотип — просто текст без іконки/брендингу
- Кнопки без ripple-ефекту та мікроанімацій
- Немає градієнтного підсвічування на активних елементах
- Footer абсолютно мінімальний (лише копірайт), немає колонок з посиланнями

#### Каталог:

- Немає сортування (за ціною, рейтингом, новизною)
- Немає фільтрів по ціновому діапазону (range slider)
- Немає перемикача grid/list вигляду
- Немає кнопки "В обране" (wishlist) на картці
- Немає кнопки "Швидко в кошик" при hover на картці
- Немає breadcrumbs (хлібні крихти)
- Немає кількості знайдених товарів
- Немає пагінації або infinite scroll

#### Картка товару / Модалка:

- Немає галереї зображень (zoom, swipe)
- Немає лічильника кількості товару (+/-)
- Немає кнопки "В обране / Wishlist"
- Немає бейджів ("Хіт", "Новинка", "Знижка -20%")
- Немає блоку "Схожі товари"
- Немає індикатора залишку ("Залишилось 3 шт!")

#### Хедер:

- Немає рядка пошуку з автопідказками (autocomplete)
- Немає мобільного бургер-меню
- Немає швидкого мінікошика при hover на іконку
- Логотип без іконки

#### Профіль та замовлення:

- Немає трекінгу замовлення (timeline статусів)
- Немає сторінки успіху після оплати
- Немає кнопки "Повторити замовлення"

### БЛОК 3. ГОЛОВНА СТОРІНКА (ПОВНИЙ РЕДИЗАЙН)

**Крок 1.** Замінити статичний банер на автоматичний слайдер (Swiper.js або власна реалізація) з 3–4 слайдами. Кожен слайд: унікальний градієнт, заголовок, підзаголовок, CTA-кнопка та декоративне зображення справа.

**Крок 2.** Навігація слайдера: точки знизу + стрілки по боках. Автозміна кожні 5 секунд.

**Крок 3.** На кожному слайді — паралакс-ефект для декоративного зображення (легке зміщення при русі мишки через JS `mousemove`).

**Крок 4.** Додати "хвилясту" нижню границю hero-секції (`clip-path: ellipse(100% 90% at 50% 0%)`) для плавного переходу до наступної секції.

**Крок 5.** Під слайдером — рядок із 4 блоками-перевагами: 🚚 Доставка по Україні | 🔒 Безпечна оплата | ✅ Гарантія якості | 💬 Підтримка 24/7. Кожен блок — іконка + текст, фон `var(--color-surface)`, border-radius 12px.

### БЛОК 6. КАТАЛОГ (РОЗШИРЕННЯ)

### БЛОК 7. АНІМАЦІЇ ТА МІКРОВЗАЄМОДІЇ

#### 7.1. Scroll-triggered анімації

**Крок 1.** Підключити `IntersectionObserver` API (або бібліотеку AOS.js).

**Крок 2.** При появі елементів у viewport — застосовувати CSS-класи:

```css
.fade-in-up {
  opacity: 0;
  transform: translateY(30px);
  transition: opacity 0.6s ease, transform 0.6s ease;
}
.fade-in-up.visible {
  opacity: 1;
  transform: translateY(0);
}
```

**Крок 3.** Застосувати до: секцій головної сторінки, карток товарів (stagger delay 0.05s × index), блоків "Як це працює", статистики.

#### 7.2. Ripple-ефект на кнопках

**Крок 1.** Додати JS обробник click на всі `.btn`:

```js
btn.addEventListener('click', e => {
  const ripple = document.createElement('span');
  ripple.classList.add('ripple');
  ripple.style.left = (e.clientX - btn.getBoundingClientRect().left) + 'px';
  ripple.style.top = (e.clientY - btn.getBoundingClientRect().top) + 'px';
  btn.appendChild(ripple);
  setTimeout(() => ripple.remove(), 600);
});
```

#### 7.3. Page transitions

**Крок 1.** При переході між сторінками — плавний fade: поточна сторінка "затухає" (opacity 0, 200ms), нова — "з'являється" (opacity 1, 200ms). Реалізується через CSS-клас на `<main>` при навігації.

#### 7.4. Кнопка "Нагору"

**Крок 1.** При скролі > 500px — у правому нижньому куті з'являється кнопка (floating action button) ↑.

**Крок 2.** При кліку — плавний скрол на початок (`scrollTo({top: 0, behavior: 'smooth'})`).

#### 7.5. Toast-нотифікації

**Крок 1.** Замість браузерних `alert()` — красиві toast-повідомлення у правому нижньому куті: "✓ Товар додано до кошика", "✓ Профіль оновлено", "✗ Помилка" тощо.

**Крок 2.** Toast з'являється знизу (`translateY(100px) → 0`), автоматично зникає через 3 секунди з прогрес-баром.
